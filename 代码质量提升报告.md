### **问题3: 线程退出的"暴力美学"** ✅ 已修复

#### **原问题**
```python
# ❌ 旧代码：简单的join()可能导致线程挂起
def stop_capture(self):
    self.is_capturing = False
    self.capture_thread.join(timeout=2)  # 可能不够
```

#### **修复方案**
```python
# ✅ 新代码：优化的stop_filter机制
def stop_filter(pkt):
    """优先级明确的停止过滤器"""
    # 优先级1: 用户主动停止
    if not self.is_capturing:
        return True

    # 优先级2: 发现设备
    if device_found:
        return True

    # 优先级3: 超时检查
    if elapsed >= duration:
        return True

    return False
```

#### **改进成果**
- ✅ 优先级明确：用户停止 > 发现设备 > 超时
- ✅ 主动告诉内核停止抓包，而不是被动等待
- ✅ 避免线程挂起和资源泄露
