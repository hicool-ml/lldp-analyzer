## 🔧 **厂商差异修复方案**

### **修复1: Feature层统一TLV安全访问**
```python
def extract_features(device) -> PortFeatures:
    """修复: 统一处理TLV缺失，建立厂商差异容错机制"""

    # 🔥 关键修复: Ruijie等厂商可能不发送Capabilities TLV
    caps_obj = safe_get(device, 'capabilities')

    # 🔥 统一容错: 确保caps永远是list，绝不会是None
    if caps_obj and hasattr(caps_obj, 'get_all_capabilities'):
        try:
            all_caps = caps_obj.get_all_capabilities()
            capabilities_list = all_caps if all_caps else []
        except:
            capabilities_list = []
    else:
        capabilities_list = []

    # 🔥 安全的特征提取: 基于list而不是对象
    features.is_router = "Router" in capabilities_list  # 安全！
    features.is_bridge = "Bridge" in capabilities_list  # 安全！
```

**关键创新**: `caps`永远是`list`，绝不可能是`None`

### **修复2: macphy_config缺失处理**
```python
# 🔥 关键修复: Ruijie等厂商可能不发送macphy_config
macphy = safe_get(device, 'macphy_config')

# 🔥 安全的速率提取: 处理macphy为None的情况
if macphy:
    speed = safe_get(macphy, 'speed', '')
    if speed:  # 确保speed不是None
        if '10G' in speed or '25G' in speed:
            features.speed_10g_plus = True
```

**关键创新**: 多重None检查，确保安全

### **修复3: format_capabilities异常处理**
```python
def format_capabilities(device) -> str:
    """修复: 安全处理Ruijie等厂商缺失Capabilities TLV"""
    caps = safe_get(device, 'capabilities')
    if not caps:
        return "未知"

    # 🔥 安全检查: get_all_capabilities可能返回None或抛出异常
    try:
        all_caps = caps.get_all_capabilities()
        if all_caps and isinstance(all_caps, list):
            return " / ".join(all_caps)
        else:
            return "未知"
    except (AttributeError, TypeError):
        return "未知"
```

**关键创新**: 异常捕获，防止崩溃
