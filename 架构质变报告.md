## 🔥 **架构质变：Feature + Rule引擎**

### **新架构层次**

```
TLV原始数据 → Feature抽象层 → 规则引擎 → 二次推断 → PortRole/DeviceType
```

#### **第1层：Feature抽象层** 🎯
```python
@dataclass
class PortFeatures:
    # VLAN特征
    has_port_vlan: bool = False
    has_protocol_vlan: bool = False
    port_vlan_tagged: bool = False
    
    # 链路聚合特征
    is_aggregated: bool = False
    aggregation_id: Optional[int] = None
    
    # MTU特征
    high_mtu: bool = False          # MTU > 2000
    jumbo_frame: bool = False       # MTU > 9000
    
    # PoE特征
    has_poe: bool = False
    
    # 设备能力特征
    is_router: bool = False
    is_bridge: bool = False
    is_wlan: bool = False
    
    # 速率特征
    speed_1g_plus: bool = False
    speed_10g_plus: bool = False
    
    # ... 共17个语义特征
```

**关键创新**：将TLV原始数据转换为结构化的语义特征集合

#### **第2层：DeviceType推断** 📱
```python
def infer_device_type(features: PortFeatures) -> DeviceType:
    # 绝对规则
    if features.has_poe and features.is_wlan:
        return DeviceType.ACCESS_POINT
    
    if features.has_poe and not features.is_wlan:
        return DeviceType.IP_PHONE
    
    if features.is_router:
        return DeviceType.ROUTER
    
    # 设备类型会反向影响PortRole推断
```

#### **第3层：规则优先级引擎** ⚖️
```python
def run_priority_rules(features: PortFeatures, device_type: DeviceType) -> Optional[PortRole]:
    # 优先级1：绝对规则（直接返回）
    
    # 规则1：链路聚合 = UPLINK_LAG（无条件）
    if features.is_aggregated:
        return PortRole.UPLINK_LAG
    
    # 规则2：Protocol VLAN存在 = TRUNK（无条件）
    if features.has_protocol_vlan:
        return PortRole.TRUNK_NATIVE
    
    # 规则3：高MTU + 1G+ = UPLINK/Storage
    if features.high_mtu and features.speed_1g_plus:
        return PortRole.STORAGE_NETWORK
    
    # 设备类型修正规则
    if device_type == DeviceType.ACCESS_POINT:
        return PortRole.ACCESS_WIRELESS
    
    # 无绝对规则，返回None继续推断
    return None
```

**关键创新**：强规则直接返回，不参与打分

#### **第4层：二次推断** 🧠
```python
def run_secondary_inference(features: PortFeatures, semantic_reasons: Set[str]) -> PortRole:
    # 特征组合推断
    
    # 组合1：PoE + 低速率 = 终端接入
    if features.has_poe and not features.speed_1g_plus:
        semantic_reasons.add("PoE_LowSpeed_Terminal")
        return PortRole.ACCESS_TERMINAL
    
    # 组合2：高速率 + 桥接 = 核心设备
    if features.speed_10g_plus and features.is_bridge:
        semantic_reasons.add("HighSpeed_Bridge_Core")
        return PortRole.CORE_DISTRIBUTION
    
    # 语义原因集合参与推断
```

**关键创新**：特征相互影响，reasons作为语义特征集合

## 📈 **对比：原架构 vs 新架构**

### **推断逻辑对比**

#### **原架构：线性加分**
```python
# 问题：PoE + 百兆 + protocol_vlan 被判成 Terminal
score = 0
if has_poe: score += 50
if speed_100m: score += 20  
if protocol_vlan: score += 40  # 应该是Trunk，但被PoE分数压倒
# 结果：错误推断
```

#### **新架构：规则优先级**
```python
# Protocol VLAN = 绝对规则，直接返回Trunk
if features.has_protocol_vlan:
    return PortRole.TRUNK_NATIVE  # 正确推断，不受其他特征影响
```

### **DeviceType参与对比**

#### **原架构：DeviceType独立推断**
```python
# 问题：DeviceType和PortRole互不影响
device_type = infer_device_type(device)  # 独立推断
port_role = infer_port_role(device)      # 独立推断
# AP设备可能被错误判成Trunk
```

#### **新架构：DeviceType修正PortRole**
```python
# DeviceType参与PortRole推断
device_type = infer_device_type(features, device)

if device_type == DeviceType.ACCESS_POINT:
    return PortRole.ACCESS_WIRELESS  # AP一定是Access，无论VLAN

if device_type == DeviceType.ROUTER:
    return PortRole.UPLINK_SINGLE     # Router一定是Uplink
```

### **reasons作用对比**

#### **原架构：reasons只是展示**
```python
reasons = []
if mtu > 1500:
    reasons.append("MTU > 1500")  # 只是文本展示
# 问题：这个信息没有参与二次推断
```

#### **新架构：reasons参与推断**
```python
semantic_reasons = set()

# 特征组合推断
if "MTU > 1500" in semantic_reasons and "Aggregation" in semantic_reasons:
    role = "Uplink-Core"  # reasons参与二次推断

# reasons现在有语义价值
semantic_reasons.add("PoE_LowSpeed_Terminal")
```
